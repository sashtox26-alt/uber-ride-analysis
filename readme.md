# Анализ данных поездок сервиса ride-hailing

## Описание проекта

В данном проекте проводится анализ данных поездок сервиса ride-hailing.  
Цель анализа — выявить закономерности спроса, определить причины отмен поездок и оценить эффективность работы сервиса в разные периоды времени.

Проект демонстрирует полный аналитический pipeline:

- загрузка и очистка данных
- создание новых признаков
- исследовательский анализ данных (EDA)
- анализ операционных показателей сервиса
- формирование бизнес-выводов

Анализ выполнен в среде **Jupyter Notebook** с использованием Python.

---

# Датасет

Датасет содержит информацию о **150 000 поездках** и включает следующие данные:

- дата и время заказа
- тип автомобиля
- точки посадки и высадки
- статус выполнения поездки
- стоимость и расстояние поездки
- время ожидания автомобиля
- причины отмен со стороны клиента и водителя
- рейтинги водителей и клиентов
- способ оплаты

# Этапы анализа

## 1. Data Loading
Загрузка датасета и первичный обзор данных.

## 2. Data Cleaning
Проверка типов данных, анализ пропусков и удаление дубликатов.

## 3. Feature Engineering
Создание дополнительных признаков для анализа:

- `datetime` — объединение даты и времени
- `hour` — час поездки
- `weekday` — день недели
- `month` — месяц
- `is_weekend` — признак выходного дня
- `is_completed` — флаг завершённой поездки
- `is_cancelled` — флаг отменённой поездки
- `time_of_day` — категория времени суток

## 4. Exploratory Data Analysis

Базовый анализ данных:

- распределение статусов поездок
- популярность типов автомобилей
- количество поездок по часам
- распределение поездок по дням недели

# Advanced Analysis

В рамках расширенного анализа были изучены ключевые операционные показатели сервиса.

## Cancellation Rate by Vehicle Type
Анализ доли отмен поездок для различных типов автомобилей.

## Demand Heatmap
Анализ распределения спроса на поездки по часам и дням недели.

## Pickup Locations Analysis
Определение наиболее популярных точек посадки.

## Waiting Time vs Cancellation
Исследование связи между временем ожидания автомобиля и вероятностью отмены поездки.

## Cancellation Reasons
Анализ причин отмен поездок со стороны клиентов и водителей.

## No Driver Found Analysis
Анализ ситуаций, когда система не смогла найти доступного водителя.

## Demand vs Supply Imbalance
Анализ дисбаланса между спросом на поездки и доступностью водителей.

## Peak Hour Performance Analysis
Оценка работы сервиса в часы пик и анализ изменения показателей при высокой нагрузке.

# Основные выводы

В ходе анализа были выявлены следующие закономерности:

- Спрос на поездки имеет выраженные пики в дневные и вечерние часы.
- Значительная доля заказов отменяется или не выполняется.
- Время ожидания автомобиля влияет на вероятность отмены поездки.
- Некоторые точки посадки генерируют значительно больше заказов.
- В периоды повышенного спроса чаще возникают случаи **No Driver Found**, что может указывать на дефицит водителей.
